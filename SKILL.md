# 微信公众号内容生产 Skill

## 核心原则

AI不是替代内容创作者，而是把低效率内容生产变成系统。每次任务都要同时产出：

1. 一篇可发布内容。
2. 可沉淀到选题库、资料库、结构库、表达库、视觉资产库的内容资产。

默认服务健身、营养、门店经营、健身房商业观察、个人IP与本地生活服务账号；用户给出其他行业时，沿用同一套内容系统。

## 触发场景

当用户要写公众号文章、长推文、营养科普、商业洞察、热点评论、行业案例、健身房经营内容、图文卡片、自动排版、公众号预览、生成配图、推送草稿箱、搭建日更流水线时使用本 skill。

## 输入识别

先从用户信息中提取：

- 账号定位：行业、身份、业务、门店类型、个人IP方向。
- 目标读者：健身房老板、教练、会员、减脂人群、潜在客户、同行、管理者等。
- 内容类型：营养干货、商业洞察、热点事件、行业案例、方法清单、门店复盘、图文卡片、销售转化文。
- 发文目标：建立信任、教育市场、获客、复购、转介绍、表达立场、追热点、沉淀系列。
- 资料来源：用户经历、门店数据、客户问题、评论区、公开报道、研究资料、图片素材、过往文章。
- 交付形态：纯正文、公众号排版HTML、带图预览、图文卡片、草稿箱推送。
- 受众审美：默认面向 30+ 高知用户时，使用克制、精致、有生活质感的表达和排版，避免廉价营销、喊口号和粗糙大字报。

信息不足时先用合理假设推进，并标注“默认假设”。只有账号定位或发布权限完全不明且会阻塞任务时，才追问一个关键问题。

## 标准生产流程

### 1. 定内容资产

先判断本篇沉淀到哪里：

- 选题库：热点、趋势、用户变化、行业误区、经营问题、营养问题、案例复盘。
- 资料库：数据、报道、研究、客户原话、门店观察、前后对比、图片证据。
- 结构库：按内容类型选择结构，不要所有文章都写成观点文。
- 表达库：标题句式、转场句、结尾号召、账号常用词和禁用词。
- 视觉资产库：封面、正文信息图、步骤卡、对比图、结尾系列图。

### 2. 选题评分

候选选题按 1-5 分评估：

- 相关性：是否贴合账号定位和目标读者。
- 稀缺性：是否有个人经验、行业细节或反常识角度。
- 可证据化：是否能用案例、数据、观察、研究支撑。
- 可复用性：是否能形成系列。
- 可视觉化：是否适合做封面、信息图或图文卡片。
- 可传播性：是否容易被收藏、转发、评论。

优先选择“高相关 + 有证据 + 能系列化 + 能视觉化”的选题。

### 3. 研究与事实校核

涉及最新事实、价格、规则、政策、新闻、公司动态、研究结论、营养医学建议或具体数据时，必须查证。无法查证的内容写成观察或假设，不写成事实。

营养内容要避免医疗化承诺；商业洞察要区分事实、推断和建议；热点评论要避免借热点硬蹭，必须回到账号长期定位。

热点文章必须先提炼“真实热点点”再写：谁、因为什么被讨论、可核实事实是什么、用户为什么会被吸引。找不到可靠来源时，不写“热搜/刷屏/节目细节”，改写成“近期讨论/社媒话题/公开履历引发的状态管理讨论”，并把不可核实部分留给用户补充素材。

### 4. 选择结构

先根据内容类型选结构，再写正文。默认不要直接套“开头-观点-三段论-结尾”。常见选择：

- 营养干货：痛点场景 -> 误区澄清 -> 原理翻译 -> 可执行方案 -> 食谱/清单 -> 注意事项。
- 商业洞察：事件/数据 -> 关键变量 -> 行业迁移 -> 门店动作 -> 风险边界。
- 热点讨论：热点事实 -> 大众误读 -> 账号立场 -> 对本行业启发 -> 行动建议。
- 行业案例：背景 -> 关键动作 -> 结果 -> 可迁移经验 -> 不可照搬部分。
- 方法清单：适用人群 -> 分组清单 -> 操作步骤 -> 避坑提醒 -> 收藏理由。
- 图文卡片：单一主题 -> 5-9 张卡片 -> 每张一个观点 -> 封底行动。

更完整模板见 [content-structures.md](references/content-structures.md)。

### 5. 写正文

默认公众号长文 1500-3500 字；图文卡片按 5-9 页；短文按用户要求。

写作要求：

- 开头 150 字内出现具体场景、冲突、事件或问题；优先用故事、对话、生活细节激发共鸣，不要一上来讲道理。
- 每个小标题只解决一个问题。
- 每 3-5 段给一次节奏变化：案例、列表、短句、提问、用户原话、数据解释。
- 观点必须先有证据，再做判断。
- 避免“赋能、打造、闭环、生态、抓手、底层逻辑”等空泛词。
- 不用“在当今时代”“随着互联网发展”“众所周知”开头。
- 不编造数字、案例、名人观点、研究结论。
- 热点转化型文章默认按“最近热点 -> 真实共鸣场景 -> 干货方法 -> 私信咨询钩子”推进。

### 6. 人工增值层

初稿后必须补一层“人味”和“行业判断”：

- 个人观点：为什么这么判断。
- 真实细节：具体人、场景、动作、时间、对话。
- 专业翻译：把营养/经营/平台术语说成人话。
- 取舍判断：什么值得做，什么不值得做。
- 账号立场：这篇文章如何强化长期定位。

如果缺少真实经历，给用户留“待补充”插槽，不假装有故事。

### 7. 排版、配图与发布

当用户要“排版”“预览”“配图”“发到草稿箱”：

1. 先读 [layout-publish.md](references/layout-publish.md)，生成公众号友好的 HTML 或 Markdown。
2. 需要图片时读 [image-production.md](references/image-production.md)，生成封面图、正文信息图或图文卡片。
3. 如果有微信公众号 AppID/AppSecret/素材权限，按草稿箱流程上传图片并创建草稿；如果没有凭证，输出本地预览链接和草稿箱推送所需字段清单。

不要声称已推送到草稿箱，除非接口返回成功结果。

### 8. 转化钩子

当用户需要获客或服务咨询时，结尾必须预留自然钩子：

- 钩子先承接文章主题，不突兀硬广。
- 引导动作清晰：私信关键词、咨询方向、适合人群。
- 可以使用“明星同款服务”作为兴趣入口，但不能暗示与明星本人有合作或保证同等效果，除非用户提供授权事实。
- 不在正文末尾显示给第三方看的事实备注、写作备注、风险备注；这些只放在内部交付说明或草稿检查里。

## 质量门槛

输出前自检：

- 是否有明确读者，而不是写给所有人。
- 是否匹配内容类型结构，而不是所有文章一个模板。
- 是否有证据、案例、经验或专业来源。
- 是否有可执行建议，而不是空泛观点。
- 是否适合公众号阅读节奏：短段落、重点句、分隔、图片插入。
- 是否有故事、细节、对话或真实场景，避免通篇说教。
- 是否符合 30+ 高知用户的审美：克制、精致、可信、不廉价。
- 是否从热点自然过渡到干货，再到私信咨询钩子。
- 是否有视觉资产方案。
- 封面是否通过手机端首屏检查：主体完整、生活场景可代入、文字可读，不因裁切丢失关键画面。
- 是否避免虚假事实、过度承诺、标题党、医疗承诺和营销腔。
- 是否把事实备注和专业风险留在内部，不显示在公众号正文尾部。
- 是否沉淀至少一个选题、一个素材、一个结构或一个视觉模板。

低于发布标准时，先修稿再输出。
